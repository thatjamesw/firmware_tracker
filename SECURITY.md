# Security Policy

## Supported Scope

This repository is a personal firmware-tracking project. Security reports are welcome for:

- GitHub Actions workflow security
- Dependency vulnerabilities
- Data parsing logic that could cause unsafe behavior in CI/deploy

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive findings.

Report privately via GitHub security advisories:

1. Open the repository Security tab.
2. Click `Report a vulnerability`.
3. Include reproduction steps, impact, and suggested mitigation if possible.

If GitHub advisory reporting is unavailable, open a private contact request with a minimal description and request a secure follow-up channel.

## Response Expectations

- Initial acknowledgement target: within 7 days
- Fix timeline: best effort, based on severity and complexity

## Disclosure

Please allow time for validation and patching before public disclosure.
